RP 0003 1stPass [Auth]: Sign-up form accepts invalid email format and weak password.
Tester: Marylle Laguna

[Summary]
During live input form field validation on the signup page, the system fails to reject an invalid email address structure (test@com) and a weak password with multiple clicks. Instead of throwing a validation error, the system accepts the credentials and incorrectly redirects the user forward.

[Precondition]
 -	User is on the Rotten Potato Signup/Registration page.
 -	No existing account is tied to the tested input
 Software version: RP.010.004
 Software configuration: Internal Testing Environment
 Hardware specification: Windows 11 Desktop
 Network configuration: Wi-Fi

[Steps to reproduce]
  1.  Navigate to the Rotten Potato Signup/Registration page.
  2.  In the email field, enter an invalid email format missing a proper TLD (e.g., test@com)
  3.  In the password field, enter a weak password that violates basic security (e.g., 123)
  4.  Multiple click the “Signup” / Submit button


[Actual results]
  1.  The live input form field validation does not trigger any error messages for the invalid email or weak password
  2.  The form submits successfully, and the user is redirected to the next page/dashboard using the incorrect credentials

[Expected results]
  1.  Inline validation errors should appear immediately or upon submission (e.g., “Please enter a valid email address” and “Password does not meet minimum strength requirements”).
  2.  The form submission should be blocked, preventing any user redirection until valid data is provided.
